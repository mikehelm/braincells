### Key Concepts to Cover

**1. Your first conversation**
Open Telegram, message your bot:
- "What's today's date?"
- "Summarize what you know about yourself"  
- "Tell me a joke about marketing"

This isn't fluff ‚Äî it's verifying the connection works and the AI has personality.

**2. Setting up HEARTBEAT.md for the first time**
Edit `~/.openclaw/workspace/HEARTBEAT.md`:
```markdown
# Graham's Heartbeat Checklist

- Good morning check: if it's between 8am-9am, send me a brief morning greeting
- If more than 12 hours since last message, send a check-in
- Check if any cron jobs have finished recently ‚Äî if yes, report summary
```

After saving, the AI picks this up on the next 30-minute cycle.

**3. Creating a first cron job: the Weekly Recap**
```bash
openclaw cron add \
  --name "Monday Morning Brief" \
  --cron "0 8 * * 1" \
  --tz "Asia/Bangkok" \
  --session isolated \
  --message "It's Monday morning. Give Graham a brief motivational note and remind him to check his lead pipeline." \
  --announce \
  --channel telegram
```

This will run every Monday at 8am Bangkok time and send Graham a message. Change timezone to match his location.

**4. Verifying everything works**
```bash
openclaw cron list          # See all scheduled jobs
openclaw cron run <job-id>  # Force-run a job immediately to test it
openclaw gateway status      # Confirm gateway is healthy
```

**5. The magic file edit workflow (safe)**
1. Open the file in any text editor (Notepad, VS Code, TextEdit on Mac)
2. Make changes
3. Save
4. The gateway picks up changes on the next session/heartbeat (no restart needed for workspace files)
5. Test with a message: "What does your HEARTBEAT.md say?" ‚Äî AI will confirm it read the update

### Quiz ‚Äî Module 6
**Complete for up to 120 bonus Brain Cells (24 per correct answer)**

**Q1:** Graham set up a cron job for Monday mornings but it didn't run. What's the most likely issue?  
‚úÖ **Answer (any of):** Timezone mismatch (UTC vs local time); Gateway not running; Job is disabled; Cron expression is wrong. Solution: `openclaw cron list` to check status, `openclaw cron run <id>` to force-test.

**Q2:** How does OpenClaw "pick up" changes to HEARTBEAT.md?  
‚úÖ **Answer:** On the next heartbeat cycle (default: ~30 minutes), the agent reads the current version of HEARTBEAT.md. Changes take effect automatically without restarting the gateway.

**Q3:** What's the difference between a "main session" cron job and an "isolated session" cron job?  
‚úÖ **Answer:** Main session = runs in the same context as your regular conversations (sees history). Isolated = starts fresh each time (no prior context, can use different model, announces summary).

**Q4:** Graham sends his OpenClaw bot "What's on my heartbeat checklist?" and gets a blank response. What might be wrong?  
‚úÖ **Answer:** HEARTBEAT.md might be empty, the gateway might not have read it yet, or the AI may need to be prompted more specifically. Try asking: "Please read your HEARTBEAT.md file and tell me what it contains."

**Q5:** What command lists all of Graham's current cron jobs?  
‚úÖ **Answer:** `openclaw cron list`

## The Great Debate (Module 6 ‚Üí 7 interlude)

**"Exciting superpower or privacy nightmare?"**

The internet is split. Not about whether OpenClaw works ‚Äî it clearly does ‚Äî but about whether giving an AI agent access to your computer is a good idea.

**Team Superpower:**
> *"I haven't done manual work in 6 months."*
> *"A smart model with eyes and hands at a desk. You message it like a coworker."* ‚Äî @nathanclark_
> *"The first time I've felt like I'm living in the future since ChatGPT launched."* ‚Äî @davemorin

**Team Concern:**
> *"You're giving an AI agent root access to your computer and a direct line to your WhatsApp. What could possibly go wrong?"*
> *"The ClawHavoc campaign proved what happens when you trust a skill registry."*

**The self-hosted argument:**
OpenClaw's founding principle is that your data never leaves your machine. Unlike Zapier (cloud), Make (cloud), or Claude.ai (cloud), OpenClaw runs locally. Your messages, your files, your automations ‚Äî all on your hardware.

The catch: that only holds if the skills you install are clean. Which brings us back to skill verification.

**Regulatory watch:**
The EU is actively working on AI agent regulations. The US watching. The question "who is responsible when an AI agent sends an email you didn't authorize?" is becoming a real legal question.

Graham's takeaway: OpenClaw's self-hosted model is genuinely more private than every alternative. The security risks are real but manageable with good hygiene. Welcome to the frontier.
