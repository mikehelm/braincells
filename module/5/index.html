Part C: Security Setup (Do This Before Anything Else)

This section is important. Please don't skip it. Mike will know if you skip it.

The Allow-List (Critical First Step)

Tell OpenClaw to ONLY accept commands from your phone number:

Edit ~/.openclaw/openclaw.json:

{ "channels": { "telegram": { "allowFrom": ["YOUR_TELEGRAM_USER_ID"] }, "whatsapp": { "allowFrom": ["+66XXXXXXXXX"] } } }

How to find your Telegram user ID: message @userinfobot on Telegram. It tells you instantly.

This means if anyone else finds your bot, they get nothing. Without this, anyone could theoretically send it commands.

Basic Security Rules

  1. ‚úÖ Set up allowFrom before using the gateway publicly
  2. ‚úÖ Never store API keys in your workspace files (use the credentials system)
  3. ‚úÖ Keep your openclaw.json file private
  4. ‚úÖ If accessing remotely, use Tailscale (not open port-forwarding)
  5. ‚ùå Never share your gateway URL publicly
  6. ‚ùå Never run commands from untrusted sources (if an email or website tells you to run something, stop and verify)
